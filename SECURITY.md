# Security Policy

## Reporting a Vulnerability

If you discover a potential security issue in this project, we ask that you report it privately to the OpenSearch Security team.

Please include:
- A clear description of the issue and its impact
- Steps to reproduce (PoC, configuration, affected versions)
- Your contact information (optional)
- Any suggested mitigation or patch

We will acknowledge your report within **48 hours**, assess severity within **5 business days**, and aim to release fixes for critical issues within **14 days**.

## Coordinated Disclosure

We support coordinated disclosure and will work with you to publish advisories once a fix is available. If applicable, we will assign a CVE and credit you in the advisory (unless you prefer anonymity).

## Supported Versions

| Version | Status         |
|---------|----------------|
| 2.x     | Actively supported |
| 1.x     | Security fixes only |
| <1.x    | End-of-life     |

## Safe Harbor

We welcome responsible security research. If you follow this policy and report privately, we will not pursue legal action. Please avoid:
- Testing against production services
- Accessing user data
- Denial-of-service or social engineering

For more details, see our [security page](https://opensearch.org/docs/latest/security-plugin/index/) and [security.txt](https://opensearch.org/.well-known/security.txt).
