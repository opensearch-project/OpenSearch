# Design Document: Star Tree Sidecar Upgrade

## Overview

This design replaces the codec-switching approach with a sidecar architecture. The codec switch breaks on segments with soft deletes — Lucene's `SegmentDocValuesProducer` routing assumes the codec in `.si` is the codec that originally wrote the base doc values.

### Architecture Change Summary

| Aspect | Old (codec switch) | New (sidecar) |
|--------|-------------------|---------------|
| GC protection | Automatic (segment file set) | `SidecarProtectedDirectory` wrapping `Store.directory()`, returned via `Store.engineDirectory()` |
| Store contract | `store.directory()` used everywhere | `store.engineDirectory()` for engines; `store.directory()` for internal Store ops (audited) |
| Reader lifecycle | Lucene codec pipeline | CAS ref counting; `unprotect()` + delete inside `decRef()` at refCount=0 |
| Cleanup lock scope | N/A | In-memory state under `sidecarMetadataLock`; disk I/O outside lock |
| Upgrade completion | Shard-local flag cleared in finally | Transport action clears flag on ALL shards after broadcast completes |
| Crash recovery | SegmentInfos atomic commit | Generational metadata (primary) + commit data (secondary) |

## Components and Interfaces

### 1. SidecarProtectedDirectory + Store.engineDirectory()

`IndexWriter.deleteUnusedFiles()` enumerates referenced files via `SegmentInfos.files()` and deletes everything else. `setLiveCommitData()` values are opaque — Lucene does NOT parse them. The only reliable protection is intercepting `deleteFile()`.

```java
public class SidecarProtectedDirectory extends FilterDirectory {
    private final Set<String> protectedFiles; // ConcurrentHashMap.newKeySet()

    @Override
    public void deleteFile(String name) throws IOException {
        if (protectedFiles.contains(name)) return; // silently skip
        super.deleteFile(name);
    }

    public void protect(Set<String> fileNames) { protectedFiles.addAll(fileNames); }
    public void unprotect(Set<String> fileNames) { protectedFiles.removeAll(fileNames); }
    public Directory getDelegate() { return in; } // bypass for intentional deletes
}
```

#### Store Integration — engineDirectory() vs directory()

`Store.directory()` returns the `StoreDirectory` (a `FilterDirectory` wrapping `ByteSizeCachingDirectory`). This is used by internal Store operations like `cleanupAndVerify()`, `removeCorruptionMarker()`, etc.

A new `Store.engineDirectory()` method returns the `SidecarProtectedDirectory` when installed, or `Store.directory()` when not:

```java
// In Store.java:
private volatile Directory engineDirectory; // null until sidecar wrapper installed

public void installSidecarDirectory(SidecarProtectedDirectory wrapper) {
    this.engineDirectory = wrapper;
}

public Directory engineDirectory() {
    Directory ed = engineDirectory;
    return ed != null ? ed : directory();
}
```

`IndexShard.newEngineConfig()` is changed to pass `store.engineDirectory()`:

```java
// In IndexShard.newEngineConfig():
return this.engineConfigFactory.newEngineConfig(
    shardId, threadPool, indexSettings, warmer,
    store,  // Store object still passed for ref counting
    // ...
);

// In EngineConfigFactory.newEngineConfig() or EngineConfig:
// The engine calls store.engineDirectory() instead of store.directory()
// to get the directory for IndexWriter construction.
```

This ensures:
- `resetEngineToGlobalCheckpoint()` → calls `newEngineConfig()` → uses `store.engineDirectory()` → gets the wrapper ✓
- `openEngineAndRecoverFromTranslog()` → calls `newEngineConfig()` → uses `store.engineDirectory()` → gets the wrapper ✓
- All engine construction paths get the wrapper ✓

#### Store.cleanupAndVerify() Audit

`Store.cleanupAndVerify()` calls `directory.deleteFile(reason, existingFile)` on the raw `StoreDirectory`. This method deletes files not in the source metadata snapshot — it would delete sidecar files.

Fix: Patch `cleanupAndVerify()` to consult the protected set:

```java
public void cleanupAndVerify(String reason, MetadataSnapshot sourceMetadata) throws IOException {
    // ...
    for (String existingFile : directory.listAll()) {
        if (Store.isAutogenerated(existingFile) || sourceMetadata.contains(existingFile)) {
            continue;
        }
        // NEW: skip sidecar-protected files
        if (engineDirectory instanceof SidecarProtectedDirectory spd && spd.isProtected(existingFile)) {
            continue;
        }
        try {
            directory.deleteFile(reason, existingFile);
        } catch (IOException ex) { /* ... existing handling ... */ }
    }
}
```

Other Store methods that call `deleteFile()` internally (audited):
- `removeCorruptionMarker()` — deletes corruption marker files only, not segment data. Safe.
- `deleteQuiet()` — called with specific file names, not a sweep. Safe unless called with sidecar file names (unlikely).
- `trimUnsafeCommits()` — deletes old commit files. Safe.

### 2. unprotect() + deleteFile() Ordering — Inside Reader's decRef()

The previous design had a race: `unprotect()` before `decRef()` allowed IndexWriter to delete files while queries were reading. The fix: `unprotect()` + `deleteFile()` happen atomically inside the reader's deferred deletion, AFTER refCount reaches zero.

```java
// In StarTreeSidecarReader:
private void deleteFiles() {
    // Called ONLY when refCount reaches 0 AND pendingDeletion is set.
    // No queries are reading these files. NOW safe to unprotect.
    sidecarProtectedDirectory.unprotect(starTreeFiles);
    for (String file : starTreeFiles) {
        try {
            sidecarProtectedDirectory.getDelegate().deleteFile(file);
        } catch (Exception e) {
            logger.warn("Failed to delete sidecar file: " + file, e);
        }
    }
}
```

The merge cleanup callback does NOT call `unprotect()`:

```java
// Merge cleanup (runs on FLUSH thread pool):
// Phase 1: In-memory updates under sidecarMetadataLock
synchronized (sidecarMetadataLock) {
    starTreeSidecarMetadata.remove(segName);
    StarTreeSidecarReader reader = sidecarReaderCache.remove(segName);
    if (reader != null) {
        reader.markPendingDeletion();
        reader.decRef(); // release cache's reference
        // If refCount=0 → deleteFiles() runs → unprotect + delete
        // If queries hold refs → deferred until last decRef()
    }
}
// Phase 2: Disk I/O outside lock
starTreeSidecarMetadata.commit(directory);
updateCommitDataForSidecarFiles();
```

### 3. StarTreeSidecarReader — Lucene-Style CAS Reference Counting

```java
public class StarTreeSidecarReader implements StarTreeValuesProvider, Closeable {
    private final AtomicInteger refCount = new AtomicInteger(1);
    private volatile boolean pendingDeletion = false;
    private final SidecarProtectedDirectory sidecarProtectedDirectory;
    private final Set<String> starTreeFiles;

    public void incRef() {
        // CAS loop — same pattern as Lucene's AbstractRefCounted
        while (true) {
            int count = refCount.get();
            if (count <= 0) {
                throw new AlreadyClosedException("StarTreeSidecarReader already closed");
            }
            if (refCount.compareAndSet(count, count + 1)) {
                return;
            }
        }
    }

    public void decRef() throws IOException {
        int count = refCount.decrementAndGet();
        assert count >= 0 : "refCount underflow";
        if (count == 0) {
            closeInternal(); // close file handles
            if (pendingDeletion) {
                deleteFiles(); // unprotect + delete — safe, no readers active
            }
        }
    }

    public void markPendingDeletion() { pendingDeletion = true; }

    @Override
    public void close() throws IOException { decRef(); }

    private void deleteFiles() {
        sidecarProtectedDirectory.unprotect(starTreeFiles);
        for (String file : starTreeFiles) {
            try { sidecarProtectedDirectory.getDelegate().deleteFile(file); }
            catch (Exception e) { logger.warn("Failed to delete: " + file, e); }
        }
    }
}
```

### 4. Upgrade Completion Coordination — Transport Action Clears Flags

The `starTreeUpgradeInProgress` flag is shard-local. The problem: if the primary clears its flag in `finally` before replicas finish, the primary serves star tree queries while replicas fall back to normal aggregation.

Fix: The flag is NOT cleared in `IndexShard.upgradeToStarTree()`'s finally block. Instead, `TransportStarTreeUpgradeAction` clears it on all shards across all nodes after the broadcast completes. Since `TransportBroadcastByNodeAction` already routes to every node hosting target shards, the flag clearing uses the same broadcast mechanism — a second lightweight broadcast (or `TransportNodesAction`) that calls `clearStarTreeUpgradeInProgress()` on each shard on each node:

```java
// In TransportStarTreeUpgradeAction:
@Override
protected void doExecute(Task task, StarTreeUpgradeRequest request, ActionListener<StarTreeUpgradeResponse> listener) {
    // ... mapping update ...

    // Broadcast shardOperation() to all shards on all nodes
    super.doExecute(task, request, ActionListener.wrap(
        response -> {
            // ALL shard operations completed. Clear flags on ALL nodes.
            broadcastClearUpgradeFlags(request, ActionListener.wrap(
                clearResponse -> listener.onResponse(response),
                clearError -> {
                    // Flag clearing failed on some nodes — log warning, still return success.
                    // Flags will be cleared on next shard start or upgrade retry.
                    logger.warn("Failed to clear upgrade flags on some nodes", clearError);
                    listener.onResponse(response);
                }
            ));
        },
        e -> {
            // Broadcast failed. Clear flags on all reachable nodes.
            broadcastClearUpgradeFlags(request, ActionListener.wrap(
                clearResponse -> listener.onFailure(e),
                clearError -> listener.onFailure(e)
            ));
        }
    ));
}

/**
 * Broadcasts clearStarTreeUpgradeInProgress() to all shards on all nodes.
 * Uses TransportBroadcastByNodeAction (or TransportNodesAction) to reach
 * remote nodes — NOT just local indicesService iteration.
 */
private void broadcastClearUpgradeFlags(StarTreeUpgradeRequest request, ActionListener<Void> listener) {
    // Option A: Reuse TransportBroadcastByNodeAction with a "clear flag" operation type
    // Option B: Use TransportNodesAction to send a lightweight clear request to each node
    // Either way, this reaches shards on ALL nodes, not just the coordinating node.
}
```

In `IndexShard`:
```java
public int upgradeToStarTree(StarTreeField starTreeField) throws IOException, InterruptedException, TimeoutException {
    verifyActive();
    if (starTreeUpgradeInProgress.compareAndSet(false, true) == false) {
        throw new IllegalStateException("star tree upgrade already in progress");
    }
    // NOTE: flag is NOT cleared here. TransportStarTreeUpgradeAction clears it
    // after ALL shards complete, to prevent result divergence during the upgrade window.
    try {
        // ... build sidecar files, flush, restart engine ...
        return upgraded;
    } catch (Exception e) {
        // On failure, clear the flag on THIS shard so it can be retried
        starTreeUpgradeInProgress.set(false);
        throw e;
    }
    // No finally block clearing the flag — transport action handles it
}

public void clearStarTreeUpgradeInProgress() {
    starTreeUpgradeInProgress.set(false);
}
```

### 5. Crash Recovery — Metadata as Primary Source of Truth

```
On shard start:
1. Scan for _startree_sidecar_gen*.meta files → load highest valid generation
2. Delete all generation files except the highest valid one (handles crash between
   write of genN+1 and delete of genN — both may exist on disk)
3. For each segment in metadata:
   a. Check if referenced files exist on disk
   b. If files exist → valid, add to protected set
   c. If files missing → stale metadata entry, remove from metadata
4. Initialize SidecarProtectedDirectory with protected set
5. Call store.installSidecarDirectory(wrapper) BEFORE engine creation
6. Cross-check against commit data (if available):
   a. Files in commit data but NOT in metadata → incomplete build, clean up files
   b. Files in metadata but NOT in commit data → metadata committed but
      commit data flush didn't complete → files are valid (metadata is primary)
7. Open engine — IndexWriter uses store.engineDirectory() → wrapped directory
```

### Platform Note: Windows Compatibility

Sidecar files are written to `store().directory()` (the raw `StoreDirectory`). `SidecarProtectedDirectory.getDelegate()` also returns the raw `StoreDirectory`. So write and delete operations target the same directory — consistent.

On POSIX systems (Linux, macOS), `deleteFile()` on a file with open handles is safe — the file is unlinked and data remains accessible until the last handle closes. On Windows, `deleteFile()` on a file with open handles may fail with `AccessDeniedException`.

The ref-counted reader ensures `deleteFiles()` is only called after refCount reaches 0 — meaning all file handles are closed before deletion. This makes the design safe on all platforms. However, if a bug causes `deleteFiles()` to be called while handles are still open, the failure mode differs: silent success on POSIX, exception on Windows. The `deleteFiles()` method wraps each `deleteFile()` call in try-catch and logs warnings on failure.

### 6. Cleanup Lock Scope — Prevent Deadlock

```java
void onMergeComplete(List<SegmentCommitInfo> mergedSegments) {
    boolean changed = false;

    // Phase 1: In-memory updates under lock (fast, no I/O)
    synchronized (sidecarMetadataLock) {
        for (SegmentCommitInfo seg : mergedSegments) {
            if (starTreeSidecarMetadata.hasStarTreeData(seg.info.name)) {
                starTreeSidecarMetadata.remove(seg.info.name);
                StarTreeSidecarReader reader = sidecarReaderCache.remove(seg.info.name);
                if (reader != null) {
                    reader.markPendingDeletion();
                    reader.decRef(); // may trigger deleteFiles() if refCount=0
                }
                changed = true;
            }
        }
    }

    // Phase 2: Disk I/O outside lock (prevents deadlock with flush)
    if (changed) {
        starTreeSidecarMetadata.commit(directory);
        updateCommitDataForSidecarFiles(); // setLiveCommitData + flush
    }
}
```

### 7. IndexShard.upgradeToStarTree()

```java
public int upgradeToStarTree(StarTreeField starTreeField) throws IOException, InterruptedException, TimeoutException {
    verifyActive();
    if (starTreeUpgradeInProgress.compareAndSet(false, true) == false) {
        throw new IllegalStateException("star tree upgrade already in progress");
    }
    // Flag NOT cleared in finally — TransportAction clears after all shards complete
    try {
        // Phase 1: Build sidecar files (engine stays live)
        int upgraded = StarTreeUpgradeService.buildSidecarStarTreeData(
            store().directory(), starTreeField, mapperService, starTreeSidecarMetadata
        );

        // Update protected directory
        sidecarProtectedDirectory.protect(starTreeSidecarMetadata.getAllSidecarFileNames());

        // Pre-drain translog OUTSIDE blockOperations
        flush(new FlushRequest().force(true).waitIfOngoing(true));

        // Phase 2: Brief block for commit data + engine restart
        indexShardOperationPermits.blockOperations(30, TimeUnit.SECONDS, () -> {
            updateCommitDataForSidecarFiles();
            flush(new FlushRequest().force(true).waitIfOngoing(true)); // fast, translog drained
            codecServiceOverride = engineConfigFactory.newDefaultCodecService(indexSettings, mapperService, logger);
            resetEngineToGlobalCheckpoint();
        });

        return upgraded;
    } catch (Exception e) {
        starTreeUpgradeInProgress.set(false); // clear on failure so shard can be retried
        throw e;
    }
}
```

## Correctness Properties

### Property 1: All engine construction paths use the wrapped directory

*For any* call to `newEngineConfig()` on an `IndexShard` where `store.engineDirectory()` returns a `SidecarProtectedDirectory`, the resulting `EngineConfig` SHALL use the wrapped directory. This includes `openEngineAndRecoverFromTranslog()`, `resetEngineToGlobalCheckpoint()`, and all recovery paths.

### Property 2: Store.cleanupAndVerify() does not delete protected files

*For any* sidecar file in the `SidecarProtectedDirectory`'s protected set, `Store.cleanupAndVerify()` SHALL skip that file during its sweep.

### Property 3: Files stay protected until last reader closes

*For any* sidecar reader with `pendingDeletion=true` and `refCount > 0`, the sidecar files SHALL remain in the protected set. `unprotect()` happens only inside `deleteFiles()` at refCount=0.

### Property 4: No star tree acceleration until all shards complete upgrade

*For any* shard where `starTreeUpgradeInProgress` is true, `isStarTreeSupported()` returns false. The flag is cleared by `TransportStarTreeUpgradeAction` only after ALL shard upgrades return.

### Property 5: Crash recovery uses metadata as primary source of truth

*For any* crash, on recovery: metadata file exists + files on disk → valid, protect. Metadata exists + files missing → stale, clean up. No metadata → clean state.

### Property 6: Cleanup lock never held during flush

*For any* merge cleanup, `sidecarMetadataLock` is released before disk I/O.

## Error Handling

### Stale Sidecar Entry at Query Time

`StarTreeSidecarReader` constructor throws `FileNotFoundException` → `StarTreeQueryHelper` catches, removes stale entry, returns null → normal aggregation fallback.

### Concurrent Reader + Cleanup Race

Cleanup: `markPendingDeletion()` + `decRef()`. Active queries hold refs → files stay protected + open. Last `decRef()` → `unprotect()` + delete. No `FileNotFoundException` for in-flight queries.

### Upgrade Failure on One Shard

Transport action clears flags on all shards (including successful ones). Failed shard's flag was already cleared in the catch block. Failed shard can be retried independently.

## Testing Strategy

### Unit Tests

1. `Store.engineDirectory()` returns wrapper when installed, raw directory when not
2. `Store.cleanupAndVerify()` skips protected files
3. `SidecarProtectedDirectory.deleteFile()` no-op for protected, passthrough for unprotected
4. CAS ref counting: `incRef` on closed throws `AlreadyClosedException`
5. `unprotect()` happens inside `deleteFiles()` at refCount=0, not before `decRef()`
6. Cleanup lock scope: lock NOT held during commit/flush
7. Crash recovery: metadata primary, commit data secondary

1. IndexWriter GC protection: `deleteUnusedFiles()` + crash recovery don't delete sidecar files
2. `cleanupAndVerify()` doesn't delete sidecar files
